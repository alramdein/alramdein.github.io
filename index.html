Data Warehouse Project

This Data Warehouse Project enables real-time sales and inventory data processing and classification to improve business intelligence. By leveraging Kafka, Windmill, ClickHouse, and MongoDB, it ensures seamless data ingestion, transformation, and storage. Change Data Capture (CDC) streams transactional updates into Kafka, triggering workflows that cleans and preprocess the data. Clean data is stored for analytics, while dirty data is tracked and re-processed using an AI-based classifier, with low-confidence cases flagged for review. This scalable and automated pipeline enhances data accuracy, provides real-time insights, and supports better decision-making.

Tech Stack: Typescript, Kafka, CDC, Windmill, ClickHouse, MongoDB, Python, AWS S3

AI Sales Forecasting

The project aims to improve forecasting accuracy by experimenting with different algorithms and incorporating relevant features like discounts and quantities.
